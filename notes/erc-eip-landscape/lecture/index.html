Ethereum Improvement Proposals (EIPs)

EIPs

  • Standards for the Ethereum platform, including core protocol specifications, client APIs, and contract standards.
  • Design document providing information on Ethereum community,rationale of feature for ethereum and its concise technical specification.
  • Author has to gather consensus within community and document dissenting opinions.

EIP Rationale (Why?)

  • Required to propose new features, gather input from community and document design decisions that have gone into Ethereum.
  • Also to track the progress of implementation by EIP implementers.

EIP classification

  1. Standard track EIP : (e.g. ERC-20)
  2. Meta EIP : (e.g. EIP-1)
  3. Informational EIP : (e.g. EIP-1470)

Standard track EIP

Standard Track EIP

  • A change that affects the interoperability of applications using Ethereum such as change to:
    • network protocol
    • block or transaction validity rules
    • proposed application standards/conventions

Standard track EIP

  • Categories of standard track EIPs
    • Core (e.g. EIP5, EIP101)
    • Networking (e.g. EIP8)
    • Interface (e.g. EIP6)
    • ERC (e.g. ERC20)

Standard track EIP

  • Core
    • consensus fork (e.g. EIP5, EIP101)
    • no consensus critical change
    • "core dev" discussions (e.g. the miner/node strategy changes 2, 3, and 4 of EIP86)

Standard track EIP

  • Networking
    • devp2p (e.g. EIP8) and light ethereum subprotocol
    • whisper and swarm and protocol improvements

Standard track EIP

  • Interface
    • client API/RPC specifications
    • method names and contract ABIs
  • The label “interface” aligns with the interfaces repo and discussion occurs in that repository before an EIP is submitted to the EIPs repository.

Standard track EIP

  • ERC
    • Application-level standards and conventions
    • token standards (ERC20), name registries (ERC137), URI schemes (ERC681), library/package formats (EIP190), and wallet formats (EIP85).

Meta EIP

Meta EIP

  • Procedures and guidelines (Process EIPs)
  • No to the Ethereum protocol or codebase
  • Require community consensus
  • More than recommendations
  • Decision-making process
  • Changes to the tools or environment.

Informational EIP

Informational EIP

  • Ethereum design issue
  • General guidelines/information to the community
  • No new feature proposal nor consensus/recommendation proposals
  • Users and implementers are free to ignore.

EIP Work Flow

EIP Work flow

  • Sherpherding an EIP
    • EIP author : vet your idea (should be original)
    • Invite feedback from community (reviewers and editors)
    • Leverage interest with implementation as well as attracting parties.

EIP Work Flow

  • Core EIPs
    • provide client implementation or convince client
    • AllCoreDevs call ("rough consensus") : for attracting client implementers to review AllCoreDevs agenda GitHub Issue

EIP Process

EIP Process

  • non-Core EIP
``` [ WIP ] -> [ DRAFT ] -> [ LAST CALL ] -> [ FINAL ] ```

EIP Process

  • Core EIP
``` [ IDEA ] -> [ DRAFT ] -> [ LAST CALL ] -> [ ACCEPTED ] -> [ FINAL ] ```
